# Отчёт о проведённом тестировании

## Краткое описание

В рамках тестирования веб-сервиса покупки тура были разработаны и выполнены автоматизированные UI-тесты.

Тестирование проводилось для проверки корректности работы формы оплаты и формы покупки в кредит, а также корректности записи данных в базу данных.

## Количество тест-кейсов

Всего было разработано и выполнено:

* UI тесты — 41 (1 тест на проверку открытия формы страницы)
* тесты проверки базы данных — 4

Общее количество тестов: **45**

## Результаты тестирования

Успешно выполнено: **25**

Неуспешно: **20**

Процент успешных тестов: **55,55%**

## Найденные дефекты

Все найденные дефекты относятся как к странице "Купить", так и к странице "Купить в кредит"

1. Карта **4444 4444 4444 4442** обрабатывается как APPROVED, хотя должна иметь статус DECLINED (В покупке и кредите).

2. Поле месяца принимает значение **00**, хотя это некорректный формат.

3. Поле владельца карты принимает кириллицу.

4. Поле владельца карты принимает имя без фамилии.

5. Поле владельца не имеет ограничений на количество вводимых символов.

6. Система принимает владельца с именем состоящем из цифр.

7. Поле CVC принимает значение **000**, хотя это некорректный формат.

8. На странице сайта с ошибкой написано название страны.
## Рекомендации

* исправить обработку карты 4442
* добавить валидацию месяца (01–12)
* ограничить ввод имени владельца карты только латиницей
* улучшить обработку ошибок формы
* добавить валидацию поля владельца (обязательно имя и фамилия, ограничение в количестве символов 30-40)
* исправить название страны на главной странице
* добавить валидацию CVC
