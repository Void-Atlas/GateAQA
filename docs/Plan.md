# План автоматизации тестирования

## Объект тестирования

Веб-сервис покупки тура.

Сервис предоставляет два способа оплаты:

* Купить
* Купить в кредит

Оплата производится банковской картой.

## Цели тестирования

Проверить корректность работы формы оплаты и формы покупки в кредит, а также корректность записи данных в базу данных.

## Задачи тестирования

1. Проверка UI формы оплаты
2. Проверка обработки различных типов карт
3. Проверка валидации полей формы
4. Проверка обработки ошибок
5. Проверка записи данных в базу данных
6. Формирование отчётов тестирования

## Инструменты автоматизации

В проекте используются следующие инструменты:

* Java 11
* Gradle
* JUnit 5
* Selenide
* Allure
* Docker
* MySQL
